Language Basics

Annotations

Annotations or comments made in code should be written with # on each side of the comment or anotation.

This works for both single lines and more. All text or characters within the #...# would be ignored.

Examples:


                        # This is a comment #
                        
                        # This part of the code calls the function
                        which is then used to give the result#

Variables

Variables are used to store data values. They are declared using the umbrella term VARIABLES: Variables: or VAR: followed by variable declarations.

This provides the relief from scanning through out a file for any declared variables, as all the variables to be used - regardless whether it is empty or not- are declared at the start of a program.

Variable Declaration

Variables declarations in LangX are slightly different to many other languages as variable names generally tend to be nouns. For this reason, variable names must always have the KEYWORD: The or the just before the variable names. 

VAR:
                        The num is 3
                        the name is "John"

Variable Naming

  • Variable names can contain letters, numbers, and underscores
  • Variable names are case-sensitive
  • Use descriptive names that explain the purpose of the variable
  • LangX supports natural language-style variable names

Variable Types

LangX supports several data types:

  • Numbers: Integers and floating-point numbers
  • Strings: Text enclosed in double quotes
  • Booleans: true or false values
  • Collections: Arrays and objects

Data Types

LangX is a dynamically typed language, meaning variables can hold values of any type.

Numbers

VAR:
                        the age is 25
                        the price is 99.99
                        the count is 1000

Strings

VAR:
                        the name is "LangX"
                        the message is "Hello, World!"
                        the path is "/usr/local/bin"

Booleans

VAR:
                        the isReady is true
                        the hasError is false

Arrays

VAR:
                        the numbers is [1, 2, 3, 4, 5]
                        the names is ["Alice", "Bob", "Charlie"]

Input/Output

LangX ...............

input the name

Operators

LangX supports various operators for performing operations on values.

Arithmetic Operators

VAR:
                        the sum is 2 + 3          # Addition: 5
                        the difference is 10 - 4   # Subtraction: 6
                        the product is 3 * 4       # Multiplication: 12
                        the quotient is 15 / 3     # Division: 5
                        the power is 2 ^ 3         # Exponentiation: 8
                        the remainder is 10 % 3    # Modulo: 1

Comparison Operators

VAR:
                        the isEqual is 5 == 5      # Equal to: true
                        the isNotEqual is 5 != 3   # Not equal to: true
                        the isGreater is 10 > 5    # Greater than: true
                        the isLess is 3 < 5        # Less than: true
                        the isGreaterEqual is 5 >= 5  # Greater than or equal: true
                        the isLessEqual is 3 <= 5     # Less than or equal: true

Logical Operators

VAR:
                        the result1 is true and false   # Logical AND: false
                        the result2 is true or false    # Logical OR: true
                        the result3 is not true         # Logical NOT: false

String Operations

VAR:
                        the greeting is "Hello" + " " + "World"  # Concatenation: "Hello World"
                        the name is "john"
                        the upperName is to adjust the name to CAPS  # Uppercase conversion

Control Flow

Conditionals

Conditional statements allow you to execute different code blocks based on conditions.

If Statements

Use if statements to execute code when a condition is true:

VAR:
                            the age is 18
                            
                            IF the age is greater than 16:
                                write "You can drive"
                            END IF

If-Else Statements

Add an else clause to handle the case when the condition is false:

VAR:
                            the temperature is 25
                            
                            IF the temperature is greater than 20:
                                write "It's warm outside"
                            ELSE:
                                write "It's cool outside"
                            END IF

Else-If Statements

Chain multiple conditions using else-if:

VAR:
                            the score is 85
                            
                            IF the score is greater than 90:
                                write "Grade: A"
                            ELSE IF the score is greater than 80:
                                write "Grade: B"
                            ELSE IF the score is greater than 70:
                                write "Grade: C"
                            ELSE:
                                write "Grade: F"
                            END IF

Loops

Loops allow you to execute a block of code repeatedly.

For Loops

Iterate over a range of values:

FOR the i from 1 to 10:
                                write the i
                            END FOR

For-Each Loops

Iterate over elements in a collection:

VAR:
                            the names is ["Alice", "Bob", "Charlie"]
                            
                            FOR EACH the name in the names:
                                write the name
                            END FOR

While Loops

Execute code while a condition is true:

VAR:
                            the count is 0
                            
                            WHILE the count is less than 5:
                                write the count
                                the count is the count + 1
                            END WHILE

Loop Control

Use break to exit a loop early, and continue to skip to the next iteration: 

FOR the i from 1 to 10:
                                IF the i is equal to 5:
                                    break
                                END IF
                                write the i
                            END FOR

Functions

Functions allow you to organize code into reusable blocks.

Defining Functions

Create a function using the func keyword:

FUNC greet(the name):
                                return "Hello, " + the name
                            END FUNC

Calling Functions

Invoke a function by its name and pass arguments:

VAR:
                            the message is greet("Alice")
                            write the message  # Output: "Hello, Alice"

Function Parameters

Functions can accept multiple parameters:

FUNC add(the a, the b):
                                return the a + the b
                            END FUNC
                            
                            VAR:
                            the result is add(5, 3)  # Returns 8

Return Values

Functions can return values using the return keyword:

FUNC multiply(the x, the y):
                                return the x * the y
                            END FUNC
                            
                            VAR:
                            the product is multiply(4, 5)  # Returns 20

Built-in Functions

LangX provides several built-in functions:

  • write - Output text to console
  • input - Read input from user
  • to adjust - String manipulation
  • the original value of - Get original variable value

Advanced Topics

Modules

Modules allow you to organize code into separate files and reuse functionality across your program.

Creating Modules

Create a module by defining functions and variables in a separate file:

# math_utils.langx
                            FUNC add(the a, the b):
                                return the a + the b
                            END FUNC
                            
                            FUNC subtract(the a, the b):
                                return the a - the b
                            END FUNC

Importing Modules

Use the import keyword to include module functionality:

IMPORT math_utils
                            
                            VAR:
                            the sum is math_utils.add(5, 3)
                            the diff is math_utils.subtract(10, 4)

Module Organization

Organize your modules in a logical structure:

  • Group related functionality together
  • Use clear, descriptive module names
  • Document your modules with comments
  • Export only what's necessary

Classes

Classes provide a way to create objects with properties and methods.

Defining Classes

Create a class using the class keyword:

CLASS Person:
                                VAR:
                                the name is ""
                                the age is 0
                                
                                FUNC init(the name, the age):
                                    the self.name is the name
                                    the self.age is the age
                                END FUNC
                                
                                FUNC greet():
                                    return "Hello, I'm " + the self.name
                                END FUNC
                            END CLASS

Creating Objects

Instantiate a class to create an object:

VAR:
                            the person is new Person("Alice", 30)
                            write the person.greet()  # Output: "Hello, I'm Alice"

Class Methods

Classes can have methods that operate on instance data:

CLASS Rectangle:
                                VAR:
                                the width is 0
                                the height is 0
                                
                                FUNC area():
                                    return the self.width * the self.height
                                END FUNC
                                
                                FUNC perimeter():
                                    return 2 * (the self.width + the self.height)
                                END FUNC
                            END CLASS

Inheritance

Create derived classes that extend base classes:

CLASS Animal:
                                VAR:
                                the name is ""
                                
                                FUNC speak():
                                    return "Some sound"
                                END FUNC
                            END CLASS
                            
                            CLASS Dog extends Animal:
                                FUNC speak():
                                    return "Woof!"
                                END FUNC
                            END CLASS

Error Handling

Error handling allows you to gracefully handle exceptions and errors in your code.

Try-Catch Blocks

Use try-catch blocks to handle potential errors:

TRY:
                                VAR:
                                the result is 10 / 0
                                write the result
                            CATCH the error:
                                write "An error occurred: " + the error.message
                            END TRY

Throwing Errors

Throw custom errors using the throw keyword:

FUNC divide(the a, the b):
                                IF the b is equal to 0:
                                    throw "Division by zero is not allowed"
                                END IF
                                return the a / the b
                            END FUNC

Error Types

LangX supports different error types:

  • Runtime Errors: Errors that occur during program execution
  • Syntax Errors: Errors in code structure
  • Type Errors: Errors related to data types
  • Custom Errors: User-defined error messages

Best Practices

  • Always handle potential errors gracefully
  • Provide meaningful error messages
  • Log errors for debugging purposes
  • Validate input before processing
  • Use specific error handling for different scenarios
